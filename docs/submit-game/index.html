Submit a Game

Share your best build with the Gurjot community. Follow the steps below to make sure the review goes smoothly.

Launch checklist

  • Playable build

    Provide a WebGL or canvas build that runs in Chromium-based browsers without additional plugins.

  • Clear controls

    Document keyboard, mouse, touch, or controller mappings inside the game or README.

  • Optimized assets

    Keep the initial download under 30 MB. Compress textures, sprites, and audio where possible.

  • Branding

    Include a 640×360 thumbnail (thumb.svg or .png) and optional animated teaser for the catalog.

Submission flow

  1. 1. Fork & set up

    Fork the repository, clone it locally, and run the game loader with npm install followed by npm run dev.

  2. 2. Add your game

    Create a folder inside /games/<your-slug> with an index.html entry point. Update games.json with title, description, and tags.

  3. 3. Document it

    Add controls and credits to README.md inside your game folder. Optional but encouraged: add screenshots to docs/assets.

  4. 4. Open a pull request

    Follow the contributing guide, run linting/tests, then open a PR titled Add <Game Name>. Include gameplay notes and any analytics hooks.

Review timeline

Most submissions are reviewed within 3–5 business days. We check for performance regressions, accessibility issues, and content compliance.

If your build requires extra capabilities (for example, same-origin sandboxing or additional storage), mention it in the PR body so we can verify security settings.

Need a hand?

Reach out in the Discord dev channel or open a draft pull request for early feedback. We keep an eye on #dev-support throughout the week.

Questions? Email support@gurjotsgames.com and include your game slug.
